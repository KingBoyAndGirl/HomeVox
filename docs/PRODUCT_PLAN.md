## 一、产品愿景

**一句话描述**：从买房到入住，用 AI + 人工的模式，让每个用户都能在 Web 上以体素级别的自由度，设计自己未来的家。

**核心主张**：

> 户型图 → AI 解析 2D 结构 → AI 生成 3D 白模 → 体素编辑器自定义 → 人工精调 → 导出效果图 + 施工资料

**定位**：

- 不是游戏，是实用工具
- 不是 CAD 替代品，是"设计沟通工具"
- AI 做 80% 粗活，人类做 20% 精细决策

**终极交付物**：

1. 高清效果图 / VR 漫游（给自己和家人看效果）
2. 施工参考图 + 材料清单 + 预算估算（给施工队执行）

## 二、用户画像与核心场景

### 2.1 目标用户

| 角色 | 痛点 | 使用方式 |
|------|------|----------|
| **业主（核心）** | 看不懂户型图，想象不出装修效果，不知道要花多少钱 | 上传户型图 → AI 生成方案 → 拖拽装修 → 导出给施工队 |
| **设计师（辅助）** | 反复改图沟通成本高，客户说不清想要什么 | 基于 AI 生成的初稿做精调，快速迭代方案 |
| **施工方（下游）** | 业主给的参考图不切实际，缺施工细节 | 接收平台导出的标准化施工参考图 + 材料清单 |

### 2.2 核心用户旅程

```
【选房阶段】
  用户拿到户型图（图片/PDF）
      ↓
【2D 阶段】
  上传户型图 → AI 识别墙、门、窗 → 生成可编辑 2D 平面图
  用户可以修改：移动墙体、改门洞位置、调整房间尺寸
      ↓
【3D 生成】
  一键生成 3D 白模（墙体、地面、天花板、门窗开口）
      ↓
【装修设计】
  选择装修风格（AI 推荐或手动）
  从体素组件库拖入家具、电器、灯具
  用体素编辑器自定义非标物品（异形柜、定制家具）
  调整材质、颜色、灯光
      ↓
【人工精调】
  设计师介入：检查方案可行性、优化空间利用率
  生成多方案对比
      ↓
【交付】
  导出：3D 漫游链接 / 效果图 / 平面图 / 材料清单 / 预算估算
```

## 三、核心功能模块

### 3.1 户型图智能解析引擎

| 功能点 | 说明 | AI/人工 |
|--------|------|---------|
| 户型图识别 | 上传图片/PDF，AI 视觉模型识别墙体、门窗、标注 | AI |
| 尺寸推断 | 从标注文字或比例尺推断实际尺寸 | AI + 人工校验 |
| 2D 结构生成 | 输出结构化数据：房间列表、墙体坐标、门窗位置 | AI |
| 手动校正 | 用户可以在 2D 视图上拖拽修正识别错误 | 人工 |

**技术方向**：

- 视觉模型：GPT-4V / Claude Vision / 开源模型做户型图解析
- 输出格式：JSON 描述墙体线段、门洞位置、房间标注
- 关键难点：手绘户型图、低清扫描件、非标准标注

### 3.2 2D 平面编辑器

| 功能点 | 说明 |
|--------|------|
| 墙体编辑 | 拖拽移动墙体、添加/删除墙段 |
| 门窗编辑 | 在墙体上添加/移动门洞、窗户 |
| 房间标注 | 自动识别 + 手动标注房间名称、面积 |
| 尺寸标注 | 关键尺寸自动标出，可手动调整 |
| 图层管理 | 原始户型图半透明叠加，便于对照 |

### 3.3 3D 体素引擎（核心差异化）

这是产品的技术核心。"体素"的含义：

- **不是**让用户逐块砖去搭房子
- **而是**所有物体的底层都是体素数据，可以实现任意形状的自定义

**3D 生成流程**：

```
2D 平面数据 → 墙体拉伸（给定层高） → 生成基础白模
   → 门窗自动开洞
   → 地面/天花板自动铺设
   → 用户可调整层高、墙体厚度
```

**体素编辑器能力**：

| 能力 | 说明 | 典型场景 |
|------|------|----------|
| 基础体素放置 | 在 3D 空间放置指定大小的方块 | 搭柜子、做隔断 |
| 体素雕刻 | 从现有物体中挖掉一部分 | 墙上掏壁龛、柜子开槽 |
| 材质刷 | 给体素表面换材质 | 选地板颜色、墙面涂料 |
| 组件库复用 | 预制的体素组件（沙发、冰箱、门） | 快速布置 |
| 品牌型号组件 | 真实品牌产品的体素模型 | 海尔冰箱 vs 西门子冰箱 |
| 自定义导入 | 用户上传 3D 模型，自动转体素 | 特殊家具、艺术品 |

**技术方向**：

- 前端 3D 引擎：Three.js / React Three Fiber
- 体素数据格式：自定义二进制格式或 MagicaVoxel 兼容
- 体素渲染优化：贪婪网格化（Greedy Meshing）+ 实例化渲染
- LOD：远景用低精度 mesh，近景显示体素细节

### 3.4 AI 设计助手

| 功能 | 说明 |
|------|------|
| 风格推荐 | 根据户型、朝向、预算，推荐装修风格（现代/北欧/日式...） |
| 自动布局 | AI 根据房间功能自动放置家具（床、沙发、餐桌等） |
| 材质搭配 | 推荐地板、墙面、柜体颜色搭配方案 |
| 违规检查 | 检查是否存在不符合建筑规范的设计（承重墙开洞等） |
| 成本估算 | 根据材料用量和地区行情，估算装修预算 |

### 3.5 渲染与导出

| 交付物 | 格式 | 说明 |
|--------|------|------|
| 3D 漫游链接 | Web URL | 分享链接，他人在浏览器里走进你的设计 |
| 高清效果图 | PNG/JPEG | 各房间多角度渲染图 |
| 平面布局图 | PDF/PNG | 带标注的 2D 平面图 |
| 材料清单 | PDF/Excel | 地板面积、墙面积、门数量、电器清单 |
| 预算估算 | PDF/Excel | 按项目分列的预估费用 |

## 五、AI + 人工协作模式

```
┌─────────────────────────────────────────────────┐
│                  AI 负责（自动化）                  │
│                                                    │
│  1. 户型图识别 → 2D 结构 (90% 准确率目标)           │
│  2. 2D → 3D 白模自动生成                           │
│  3. 基于户型的家具自动布局                          │
│  4. 装修风格 / 材质搭配推荐                         │
│  5. 材料用量和预算自动计算                          │
│  6. 施工可行性基础检查（规范碰撞）                    │
│                                                    │
└────────────────────┬───────────────────────────────┘
                     │ AI 生成初稿
                     ▼
┌─────────────────────────────────────────────────┐
│              人工负责（决策 + 精调）                │
│                                                    │
│  1. 确认/修正 AI 的户型识别结果                     │
│  2. 选择装修风格方向                                │
│  3. 微调家具位置、替换组件                          │
│  4. 定制非标家具（体素编辑器）                       │
│  5. 设计师审核方案可行性                            │
│  6. 最终效果确认                                    │
│                                                    │
└─────────────────────────────────────────────────┘
```

**关键原则**：

- AI 给用户**选项**，不给唯一答案
- 人工只做 AI 做不到的精细决策
- 每一步 AI 输出都可以被人工覆盖

## 六、MVP 范围与迭代路线

### Phase 0：技术验证（2-3 周）

| 子任务 | 产出 | 验证标准 |

## 七、竞品分析

| 竞品 | 模式 | 优势 | 不足 |
|------|------|------|------|
| **酷家乐** | SaaS 工具 | 渲染质量高，B 端覆盖广 | 操作复杂，偏向专业设计师 |
| **住小帮** | 内容社区 + 工具 | 用户量大，案例丰富 | 2D 为主，3D 弱 |
| **好好住** | 内容社区 | 真实案例，用户黏性高 | 工具属性弱 |
| **土巴兔** | 平台（撮合） | 连接用户和装修公司 | 工具属性弱，口碑争议 |
| **SketchUp** | 通用 3D 建模 | 灵活、插件丰富 | 学习曲线高，不专门为装修设计 |
| **Minecraft** | 游戏 | 体素自由度极高 | 不是装修工具，精度低 |

**VoxHome 的差异化**：

1. **体素级自定义**：既有 Minecraft 的自由度，又有装修工具的精度
2. **AI 驱动**：户型图自动解析 + AI 布局，大幅降低使用门槛
3. **全流程覆盖**：从户型图到施工资料的完整闭环
4. **Web 优先**：无需安装，分享即协作

## 九、关键风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 户型图 AI 识别准确率不达标 | 用户体验差，人工修正成本高 | Phase 0 先做技术验证，不行就降低自动化程度 |
| 3D 体素渲染性能 | Web 端卡顿 | Greedy Meshing + LOD + Web Worker 卸载计算 |
| 体素编辑器交互复杂 | 学习曲线高 | 先做"半自动"模式（AI 生成 + 拖拽微调），不追求全手工 |
| 缺少品牌 3D 模型数据 | 组件库内容少 | 先用通用模型，后续开放用户上传 + 品牌合作 |
| 单人体力有限 | 开发周期长 | Workflow + AI Agent 并行开发，利用 delegate_task 批量推进 |
